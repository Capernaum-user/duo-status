기획 요약

Gnosis 자가성장 루프의 학습 로그가 **중복으로 붕괴**했다. 이 로그는 세션 시작 훅이 읽어 **모든 에이전트 프롬프트에 주입**(최근 6KB 캡)되므로, 중복이 캡을 다 먹으면 실제 교훈이 밀려나 **학습 신호가 노이즈가 된다**.

작업 결과

Gnosis 학습로그(`state/gnosis/reflections.md`·`skills.md`)가 같은 반려 사유/성공 문구를 무한 누적하며 붕괴한 문제를 뿌리(생산자)부터 고쳤다.

1. **`console/gnosis.py`** — `_append()`를 정규화-키 기반 중복 병합으로 재작성.
- `_normalize_key()`: `(×N)` 접미사·run id 등 8~40자 hex 토큰·중복 공백을 제거해 "가변부 빼고 본문만" 비교하는 키를 만든다.
- `_merge_entry()`: 같은 정규화 키가 이미 있으면 그 줄을 **최신 시각 + 발생횟수(×N)** 로 갱신하고(줄 추가 없음), 목록 끝(최근 활동 순)으로 옮긴다. 없으면 새 줄 추가.
- 상한 가드: 모듈 상수 `_MAX_ENTRIES = 200`(각 파일). 초과 시 "가장 오래 손대지 않은" 항목부터 제거(최근-갱신 항목은 끝으로 이동해 있으므로 앞쪽=오래된 것이 잘림).
- 파일 I/O 원자화: `_write_entries_atomic()` — `.tmp` 파일에 쓰고 `Path.replace()` 로 교체.
- `_append()` 전체가 여전히 `try/except Exception: return False` 로 감싸여 있어 **예외 비전파 계약 불변**. `record_success`/`record_failure` 시그니처·호출부는 무변경.

2. **기존 로그 압축(데이터 정리)** — `gnosis.py`와 **동일한** `_load_entries`/`_parse_entry_line`/`_merge_entry` 함수를 재사용하는 1회성 스크립트(`compact_gnosis.py`, 리포 밖 temp 경로에 작성·실행, 리포에 커밋 안 됨)로 원본을 시간순 재생(replay)하며 병합했다. 각 항목의 **원래 타임스탬프**를 그대로 사용했으므로(현재 시각 아님) "이 dedup 로직이 처음부터 있었다면 나왔을 결과"와 동일하다. 헤더 주석은 그대로 보존.
